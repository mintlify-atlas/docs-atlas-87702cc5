Creating a civilization with the TypeScript tools involves defining the civilization, creating its unique abilities via `ModifierBuilder`, binding unique content, and importing the icon.

## Complete example

Here's a full civilization definition — Dacia with a unique melee combat ability:

```typescript
import {
  ACTION_GROUP_BUNDLE,
  CivilizationBuilder,
  ImportFileBuilder,
  Mod,
  ModifierBuilder,
  TAG_TRAIT,
  TRAIT,
  COLLECTION,
  EFFECT,
  REQUIREMENT,
  UNIT_CLASS
} from "@mateicanavra/civ7-sdk";

const mod = new Mod({ id: "my-civ-mod", version: "1.0" });

// Import the civilization icon
const civIcon = new ImportFileBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_ANTIQUITY,
  content: "./assets/civ-icon.png",
  name: "civ_sym_dacia"
});

// Define the civilization
const dacia = new CivilizationBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_ANTIQUITY,

  civilization: {
    domain: "AntiquityAgeCivilizations",
    civilizationType: "CIVILIZATION_DACIA"
  },

  // Traits that affect AI behavior and gameplay classification
  civilizationTraits: [
    TRAIT.ANTIQUITY_CIV,
    TRAIT.ATTRIBUTE_MILITARISTIC,
    TRAIT.ATTRIBUTE_CULTURAL
  ],

  // Tags for categorization in selection screens
  civilizationTags: [
    TAG_TRAIT.CULTURAL,
    TAG_TRAIT.MILITARY
  ],

  icon: {
    path: `fs://game/${mod.id}/${civIcon.name}`
  },

  localizations: [{
    name: "Dacia",
    description: "Ancient kingdom in Eastern Europe known for fierce warriors and gold mines.",
    fullName: "Kingdom of Dacia",
    adjective: "Dacian",
    cityNames: ["Sarmizegetusa", "Argidava", "Buridava", "Sucidava", "Petrodava"]
  }],
});

// Define the unique ability
const dacianUprisings = new ModifierBuilder({
  modifier: {
    collection: COLLECTION.PLAYER_UNITS,
    effect: EFFECT.UNIT_ADJUST_COMBAT_STRENGTH,
    permanent: true,
    requirements: [{
      type: REQUIREMENT.UNIT_TAG_MATCHES,
      arguments: [{ name: "Tag", value: UNIT_CLASS.MELEE }]
    }],
    arguments: [{ name: "Amount", value: 5 }]
  },
  localizations: [{
    description: "Mountain Warriors: +5 Combat Strength for Melee units."
  }]
});

// Bind the ability to the civilization
dacia.bind([dacianUprisings]);

// Add everything to the mod
mod.add([dacia, civIcon]).build("./dist");
```

## Key components

| Property | Description |
|---|---|
| `civilization.civilizationType` | Unique identifier (e.g., `CIVILIZATION_DACIA`) |
| `civilization.domain` | Age category: `AntiquityAgeCivilizations`, `ExplorationAgeCivilizations`, `ModernAgeCivilizations` |
| `civilizationTraits` | Predefined traits from the `TRAIT` constant (affect AI behavior and eligibility) |
| `civilizationTags` | Tags for display in selection screens (from `TAG_TRAIT`) |
| `icon` | Path to the civilization icon |
| `localizations` | Name, description, adjective, full name, and city names |

## What `bind()` does

The `bind()` method on `CivilizationBuilder` creates associations between the civilization and its unique content:

- **`ModifierBuilder`** — adds modifiers to the civilization's trait, so the ability activates when the player plays that civ
- **`UnitBuilder`** — sets the unit's `traitType` and adds it to the civilization selection screen
- **`ConstructibleBuilder`** — sets the building's `traitType` making it civilization-exclusive
- **`ProgressionTreeBuilder`** — wires up the unique civic tree reveal modifier
- **`LeaderUnlockBuilder`** — associates a leader with the civilization

## Multiple abilities

Bind as many modifiers as you need:

```typescript
const mountainWarfare = new ModifierBuilder({
  modifier: {
    collection: COLLECTION.PLAYER_UNITS,
    effect: EFFECT.UNIT_ADJUST_COMBAT_STRENGTH,
    permanent: true,
    requirements: [{ type: REQUIREMENT.COMBAT_IS_IN_HILLS }],
    arguments: [{ name: "Amount", value: 3 }]
  },
  localizations: [{ description: "+3 Combat Strength in Hills." }]
});

const goldsmithCraft = new ModifierBuilder({
  modifier: {
    collection: COLLECTION.PLAYER_CITIES,
    effect: EFFECT.CITY_ADJUST_YIELD_FROM_TAG,
    permanent: true,
    arguments: [
      { name: "Tag", value: "PRODUCTION" },
      { name: "YieldType", value: "YIELD_GOLD" },
      { name: "Amount", value: 25 },
      { name: "Percent", value: true }
    ]
  },
  localizations: [{ description: "+25% Gold from production buildings." }]
});

dacia.bind([mountainWarfare, goldsmithCraft]);
```

## Adding unique units and buildings

```typescript
import { UnitBuilder, ConstructibleBuilder, UNIT_CLASS, CONSTRUCTIBLE_TYPE_TAG } from "@mateicanavra/civ7-sdk";

const falxWarrior = new UnitBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_ANTIQUITY,
  typeTags: [UNIT_CLASS.MELEE, UNIT_CLASS.HEAVY],
  unit: { unitType: "UNIT_DACIA_FALX_WARRIOR" },
  unitCost: { cost: 90 },
  unitStat: { combat: 35 },
  localizations: [{
    name: "Falx Warrior",
    description: "Dacian warrior wielding the curved falx blade."
  }]
});

const goldMine = new ConstructibleBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_ANTIQUITY,
  constructible: { constructibleType: "BUILDING_DACIA_GOLD_MINE" },
  building: {},
  typeTags: [CONSTRUCTIBLE_TYPE_TAG.UNIQUE, CONSTRUCTIBLE_TYPE_TAG.ECONOMY],
  constructibleYieldChanges: [{ yieldType: "YIELD_GOLD", yieldChange: 5 }],
  localizations: [{
    name: "Dacian Gold Mine",
    description: "Rich gold mines famous throughout antiquity."
  }]
});

dacia.bind([falxWarrior, goldMine]);
mod.add([dacia, civIcon, falxWarrior, goldMine]).build("./dist");
```

## Age-specific civilizations

Declare separate builder instances for each Age and connect them with `CivilizationUnlockBuilder`:

```typescript
import { CivilizationUnlockBuilder, AGE } from "@mateicanavra/civ7-sdk";

const ancientDacia = new CivilizationBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_ANTIQUITY,
  civilization: {
    domain: "AntiquityAgeCivilizations",
    civilizationType: "CIVILIZATION_DACIA_ANCIENT"
  },
  // ...
});

const explorationDacia = new CivilizationBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_EXPLORATION,
  civilization: {
    domain: "ExplorationAgeCivilizations",
    civilizationType: "CIVILIZATION_DACIA_EXPLORATION"
  },
  // ...
});

const daciaTransition = new CivilizationUnlockBuilder({
  actionGroupBundle: ACTION_GROUP_BUNDLE.AGE_EXPLORATION,
  from: {
    civilizationType: "CIVILIZATION_DACIA_ANCIENT",
    ageType: AGE.ANTIQUITY
  },
  to: {
    civilizationType: "CIVILIZATION_DACIA_EXPLORATION",
    ageType: AGE.EXPLORATION
  }
});

mod.add([ancientDacia, explorationDacia, daciaTransition]).build("./dist");
```

## What the builder generates

`CivilizationBuilder` generates six XML files in `/civilizations/civilization-type-name/`:

| File | Purpose |
|---|---|
| `current.xml` | Game-scope data (types, traits, city names, start biases) |
| `legacy.xml` | Always-loaded data for Age transition compatibility |
| `shell.xml` | Shell/frontend data (selection screen, tags, leader unlocks) |
| `icons.xml` | Icon definitions |
| `localization.xml` | English text |
| `game-effects.xml` | Modifiers for unique abilities |
