## `EFFECT`

The `EFFECT` constant contains over 500 game effect identifiers. Pass them to `ModifierBuilder.modifier.effect`.

```typescript
import { EFFECT } from '@mateicanavra/civ7-sdk';
```

### Selected yield effects

| Key | Value | Description |
|---|---|---|
| `EFFECT.PLAYER_ADJUST_YIELD` | `'EFFECT_PLAYER_ADJUST_YIELD'` | Flat yield change for the player |
| `EFFECT.CITY_ADJUST_YIELD` | `'EFFECT_CITY_ADJUST_YIELD'` | Flat yield change in a city |
| `EFFECT.CITY_ADJUST_YIELD_PER_RESOURCE` | `'EFFECT_CITY_ADJUST_YIELD_PER_RESOURCE'` | Yield scaled by local resources |
| `EFFECT.CITY_ADJUST_YIELD_PER_POPULATION` | `'EFFECT_CITY_ADJUST_YIELD_PER_POPULATION'` | Yield scaled by population |
| `EFFECT.PLAYER_ADJUST_CONSTRUCTIBLE_YIELD` | `'EFFECT_PLAYER_ADJUST_CONSTRUCTIBLE_YIELD'` | Yield change from constructibles |

### Usage example

```typescript
import { ModifierBuilder, COLLECTION, EFFECT, REQUIREMENT } from '@mateicanavra/civ7-sdk';

const modifier = new ModifierBuilder({
    modifier: {
        collection: COLLECTION.OWNER,
        effect: EFFECT.CITY_ADJUST_YIELD,
        permanent: true,
        requirements: [{
            type: REQUIREMENT.CITY_IS_CAPITAL,
        }],
        arguments: [
            { name: 'YieldType', value: 'YIELD_CULTURE' },
            { name: 'Amount', value: 3 },
        ],
    },
    localizations: [{ description: '+3 Culture in your capital' }],
});
```

## `COLLECTION`

The collection determines which subjects the modifier evaluates against.

```typescript
import { COLLECTION } from '@mateicanavra/civ7-sdk';
```

| Key | Description |
|---|---|
| `COLLECTION.OWNER` | The player (civ) that owns the entity |
| `COLLECTION.ALL_CITIES` | All cities owned by the player |
| `COLLECTION.ALL_UNITS` | All units owned by the player |
| `COLLECTION.CITY` | The specific city (used with city modifiers) |

## `REQUIREMENT`

Requirements gate when a modifier is active. They appear in `modifier.requirements`.

```typescript
import { REQUIREMENT } from '@mateicanavra/civ7-sdk';
```

### Common requirements

| Key | Value | Description |
|---|---|---|
| `REQUIREMENT.CITY_IS_CAPITAL` | `'REQUIREMENT_CITY_IS_CAPITAL'` | Only applies in the capital |
| `REQUIREMENT.CITY_IS_CITY` | `'REQUIREMENT_CITY_IS_CITY'` | Only applies in full cities (not towns) |
| `REQUIREMENT.CITY_HAS_UNIQUE_QUARTER` | `'REQUIREMENT_CITY_HAS_UNIQUE_QUARTER'` | Only when a specific unique quarter is present |
| `REQUIREMENT.PLAYER_HAS_CIVILIZATION_TRAIT` | `'REQUIREMENT_PLAYER_HAS_CIVILIZATION_TRAIT'` | Only for players with a specific civ trait |
| `REQUIREMENT.PLAYER_HAS_CIVILIZATION_OR_LEADER_TRAIT` | `'REQUIREMENT_PLAYER_HAS_CIVILIZATION_OR_LEADER_TRAIT'` | Matches civ or leader trait |
| `REQUIREMENT.PLAYER_LEADER_TYPE_MATCHES` | `'REQUIREMENT_PLAYER_LEADER_TYPE_MATCHES'` | Specific leader is playing |

### Usage with arguments

Many requirements accept `arguments` that narrow the condition:

```typescript
requirements: [
    {
        type: REQUIREMENT.CITY_HAS_UNIQUE_QUARTER,
        arguments: [{ name: 'UniqueQuarterType', value: 'QUARTER_GONDOR' }],
    },
    {
        type: REQUIREMENT.PLAYER_HAS_CIVILIZATION_OR_LEADER_TRAIT,
        arguments: [{ name: 'TraitType', value: 'TRAIT_GONDOR' }],
    },
]
```

## `REQUIREMENT_SET`

```typescript
import { REQUIREMENT_SET } from '@mateicanavra/civ7-sdk';

REQUIREMENT_SET.TEST_ALL        // All requirements must pass
REQUIREMENT_SET.TEST_LEAST_ONE  // At least one requirement must pass
```
