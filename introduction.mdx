Civ7 Modding Tools is a monorepo workspace that replaces manual XML editing with a programmatic approach to Civilization VII modding. You write TypeScript, and the SDK generates the XML files, `.modinfo`, and folder structure your mod needs.

The project started as a fork of [izica/civ7-modding-tools](https://github.com/izica/civ7-modding-tools) and has since been extended with a full TypeScript SDK, a command-line tool, plugin libraries, and this documentation site.

## Who this is for

- **Mod developers** who want type-safe, programmatic mod creation instead of editing raw XML
- **Technical modders** who prefer working in code and benefit from IntelliSense, type checking, and version control
- **Tool builders** who need reusable libraries for Civ7 data manipulation and graph analysis

## How it fits together

The workspace is organized as a monorepo:

```
civ7-modding-tools/
├── packages/
│   ├── sdk/              # @mateicanavra/civ7-sdk — TypeScript mod builders
│   ├── cli/              # @mateicanavra/civ7-cli — the `civ7` command
│   ├── config/           # Shared config and path resolution
│   └── plugins/
│       ├── plugin-files/ # Zip/unzip logic
│       └── plugin-graph/ # XML graph crawl and render
├── apps/
│   ├── docs/             # This documentation site
│   └── playground/       # Example mods and experiments
├── mods/                 # Colocated mod workspaces
└── civ.config.jsonc      # CLI configuration
```

You install the SDK in your mod project and use the CLI from the repo root to manage game data. The two tools are independent — you can use the SDK without the CLI, and vice versa.
